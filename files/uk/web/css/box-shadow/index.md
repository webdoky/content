Властивість [CSS](/uk/docs/Web/CSS) **`box-shadow`** ("тінь коробки") додає ефекти тіні навколо рамки елемента. Можна вказати кілька ефектів через кому. Тінь описується відступами за осями X та Y від елемента, радіусами розмиття та розмаху, а також кольором.

{{EmbedInteractiveExample("pages/css/box-shadow.html")}}

Властивість `box-shadow` дає змогу відкидати тінь від рамки майже будь-якого елемента. Якщо на елементі з тінню вказано {{cssxref("border-radius")}}, то тінь так само матиме заокруглені кути. Порядок нагромадження кількох тіней такий же, як для кількох [тіней тексту](/uk/docs/Web/CSS/text-shadow) (раніше описана перекриває наступні).

[Генератор box-shadow](/uk/docs/Web/CSS/CSS_Background_and_Borders/Box-shadow_generator) – інтерактивний інструмент, що дає змогу згенерувати `box-shadow`.

## Синтаксис

```css
/* Ключові значення */
box-shadow: none;

/* offset-x | offset-y | color */
box-shadow: 60px -16px teal;

/* offset-x | offset-y | blur-radius | color */
box-shadow: 10px 5px 5px black;

/* offset-x | offset-y | blur-radius | spread-radius | color */
box-shadow: 2px 2px 2px 1px rgba(0, 0, 0, 0.2);

/* inset | offset-x | offset-y | color */
box-shadow: inset 5em 1em gold;

/* Будь-яка кількість тіней, розділених комами */
box-shadow: 3px 3px red, -1em 0 0.4em olive;

/* Глобальні ключові слова */
box-shadow: inherit;
box-shadow: initial;
box-shadow: revert;
box-shadow: unset;
```

`box-shadow` можна вказати за допомогою:

- Двох, трьох або чотирьох значень [`<length>`](/uk/docs/Web/CSS/length).

  - Якщо вказані два значення, вони приймаються у якості значень `<offset-x>` та `<offset-y>` (відступи за осями X та Y).
  - Якщо вказане третє значення, воно приймається у якості `<blur-radius>` (радіус розмиття).
  - Якщо вказане четверте значення, воно приймається у якості `<spread-radius>` (радіус розмаху).

- Необов‘язково – ключове слово `inset` (вкладена).
- Необов‘язково – значення [`<color>`](#color).

Аби вказати кілька тіней, слід вказати тіні через кому.

### Значення

- `inset` (вкладена)
  - : Якщо не вказане (усталено), тінь вважається відкинутою (такою, ніби коробка піднята над рештою вмісту).
    Присутність ключового слова `inset` перетворює тінь на таку, що відкинута всередину рамки (так, ніби вміст коробки вдавлений). Внутрішні тіні зображаються всередині межі (навіть прозорої межі), поверх тла, але під вмістом.
- `<offset-x>` `<offset-y>`
  - : Це два значення {{cssxref("&lt;length&gt;")}} для опису відступу тіні. `<offset-x>` вказує горизонтальний відступ. Від‘ємні значення поміщають тінь зліва від елемента. `<offset-y>` вказує вертикальний відступ. Від‘ємні значення поміщають тінь згори від елемента. Дивіться {{cssxref("&lt;length&gt;")}} для інформації про можливі одиниці вимірювання.
    Якщо обидва значення дорівнюють `0`, тінь поміщається під елементом (і може утворювати ефект розмиття, якщо вказано значення `<blur-radius>` або `<spread-radius>`).
- `<blur-radius>` (радіус розмиття)

  - : Це третє значення {{cssxref("&lt;length&gt;")}}. Чим воно більше, тим більше розмиття, тож тінь стає більшою та світлішою. Від‘ємні значення не допускаються. Якщо не вказане, то вважається рівним `0` (край тіні – різкий). Специфікація не вміщає конкретного алгоритму того, як повинен обчислюватись радіус розмиття, втім, пояснює наступне:

    > …щоб отримати довгий, прямий край тіні, повинен бути створений перехід кольору завдовжки як відстань розмиття, перпендикулярний до і відцентрований на краю тіні, що коливається від повного кольору тіні на внутрішньому кінці радіусу до повної прозорості на зовнішньому його кінці.

- `<spread-radius>` (радіус розмаху)
  - : Це четверте значення {{cssxref("&lt;length&gt;")}}. Додатні значення змусять тінь розширюватись та збільшуватись, від‘ємні – зменшуватись. Якщо не вказане – вважається рівним `0` (Тінь матиме такий же розмір, як елемент).
- `<color>` (колір)
  - : Дивіться значення {{cssxref("&lt;color&gt;")}} для інформації про можливі ключові слова та формати значень.
    Якщо не вказане – вважається рівним до {{cssxref("&lt;color&gt;","currentColor","#currentcolor_keyword")}}.

### Інтерполяція

Кожна тінь у списку (розглядаючи `none` як список нульової довжини) інтерполюється через кольорову (як колір) компоненту, а також компоненти x, y, розмиття та (коли припустимо) розмаху (як довжина) компоненти. Для кожної тіні, якщо обидві тіні є, або обидві не є `inset`, то інтерпольована тінь мусить збігатись з вихідними тінями щодо цього. Якщо у будь-якій парі вихідних тіней одна є `inset`, а інша не є `inset`, то увесь список тіней не інтерполюється. Якщо списки тіней мають різну довжину, то коротший список в кінці заповнюється тінями, чий колір – `transparent`, усі довжини – `0`, і для котрих `inset` вказано чи не вказано – згідно з вихідним списком, поки короткий список не відповідатиме за довжиною довгому спискові.

## Приклади

### Три тіні

У цьому прикладі три тіні: внутрішня, звичайна відкинута, а також 2-піксельна тінь, що створює ефект межі (замість останньої можна було б використати {{cssxref('outline')}}).

#### HTML

```html
<blockquote>
  <q
    >Можете в мене стріляти словами,<br />
    Штрикать поглядом крізь бійниці вогню,<br />
    І вбивати своєю жорстокістю,<br />
    Але все ж, як життя, я встаю.</q
  >
  <p>&mdash; Майя Енджелоу (переклад – Таня ШерепКо)</p>
</blockquote>
```

#### CSS

```css
blockquote {
  padding: 20px;
  box-shadow:
       inset 0 -3em 3em rgba(0,0,0,0.1),
             0 0  0 2px rgb(255,255,255),
             0.3em 0.3em 1em rgba(0,0,0,0.3);
}
```

### Нуль як значення відступу та розмиття

Коли значення `x-offset`, `y-offset` та `blur` – нуль, тінь матиме контур суцільного кольору рівного розміру з усіх боків. Тіні відображаються від спідньої до верхньої, тож перша тінь лежить поверх наступних тіней. Коли значення `border-radius` – 0, як усталено, то кути тіні будуть, зрештою, кутами. Якщо ж вказана властивість `border-radius` з будь-яким іншим значенням, то кути будуть закруглені.

Додано зовнішнє поле такого ж розміру, як найширша тінь, аби пересвідчитись, що тінь не перекриває суміжні елементи й не виходить за межі контейнерної коробки. Коробкова тінь не вражає вимірів [коробкової моделі](/uk/docs/Web/CSS/CSS_Box_Model).

#### HTML

```html
<div><p>Привіт, світе</p></div>
```

#### CSS

```css
p {
  box-shadow: 0 0 0 2em #f4aab9, 0 0 0 4em #66ccff;
  margin: 4em;
  padding: 1em;
}
```
